Frequently Asked Questions

Everything you need to know about Armoryx runtime protection.

Runtime Application Self-Protection (RASP) is a security technology that runs inside an application and detects attacks in real time. Unlike perimeter defenses, RASP monitors app behavior from within — detecting tampering, hooking, debugging, and environment manipulation as it happens, and responding immediately to neutralize threats.

No. Armoryx uses binary-level wrapping technology. You upload your compiled APK, AAB, or IPA file, and our platform applies protection directly to the binary. No SDK integration, no code modifications, and no access to source code is required. This means any team — regardless of tech stack — can protect their apps.

Yes. Armoryx supports both Android (APK and AAB formats) and iOS (IPA) with platform-specific protection techniques optimized for each OS. Android protection includes DEX-level and native library hardening, while iOS protection covers Mach-O binary protection and Swift/Objective-C runtime guards.

Armoryx adds less than 3 milliseconds to app startup time and has zero measurable impact on runtime performance. Our protection operates at the native layer using highly optimized checks that consume negligible CPU and memory. App size increases by approximately 1-2MB depending on protection configuration.

Armoryx detects and responds to: code tampering and repackaging, runtime hooking (Frida, Xposed), debugging and instrumentation, root/jailbreak environments, man-in-the-middle attacks, VPN/proxy usage, screen capture and recording, emulator execution, overlay attacks, and keylogger injection. Response actions are configurable per threat type.
